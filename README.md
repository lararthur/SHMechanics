Setting up a Unity project on GitHub the right way (with `.gitignore` and Git LFS) is one of the most important things you can do as a game developer. It guarantees you will never lose your progress or bloat your repository with useless temporary files.

Here is your step-by-step guide to doing it right from the very beginning.

### Prerequisites

Before we start, make sure you have these installed on your computer:

1. **Git:** The core version control system.
2. **Git LFS:** The Large File Storage extension (you usually download and install this separately from the main Git website).
3. A **GitHub account**.

---

## Step 1: Create the `.gitignore` File (Do this FIRST)

We must tell Git to ignore Unity's massive temporary folders *before* we track any files. If you track them by accident, it is a headache to undo.

1. Open your Unity project folder on your computer (the folder containing your `Assets`, `ProjectSettings`, etc.).
2. Create a new plain text file in this root folder and name it exactly **`.gitignore`** (Make sure your OS doesn't hide the extension and accidentally name it `.gitignore.txt`).
3. Open the file in Notepad or any text editor, paste the following standard Unity ignore list into it, and save:

```text
# This prevents Git from tracking Unity's massive temporary files
/[Ll]ibrary/
/[Tt]emp/
/[Oo]bj/
/[Bb]uild/
/[Bb]uilds/
/[Ll]ogs/
/[Uu]ser[Ss]ettings/

# Memory captures and profiler files
*.pidb.meta
*.pdb.meta
*.mdb.meta
sysinfo.txt
*.apk
*.unitypackage

# Autogenerated VS/MD/Consulo solution and project files
ExportedObj/
.consulo/
*.csproj
*.unityproj
*.sln
*.suo
*.tmp
*.user
*.userprefs
*.pidb
*.booproj
*.svd
*.pdb
*.mdb
*.opendb
*.VC.db

```

---

## Step 2: Set Up Git LFS (Large File Storage)

Now we need to tell Git which file types are too bulky for standard tracking (like 3D models, audio, and large textures) so it routes them through LFS instead.

1. In that same root project folder, create another plain text file and name it exactly **`.gitattributes`**.
2. Open it, paste this list of common game development file types, and save:

```text
# 3D models
*.fbx filter=lfs diff=lfs merge=lfs -text
*.obj filter=lfs diff=lfs merge=lfs -text
*.blend filter=lfs diff=lfs merge=lfs -text

# Audio
*.mp3 filter=lfs diff=lfs merge=lfs -text
*.ogg filter=lfs diff=lfs merge=lfs -text
*.wav filter=lfs diff=lfs merge=lfs -text

# Images and Textures
*.png filter=lfs diff=lfs merge=lfs -text
*.jpg filter=lfs diff=lfs merge=lfs -text
*.jpeg filter=lfs diff=lfs merge=lfs -text
*.tga filter=lfs diff=lfs merge=lfs -text
*.tif filter=lfs diff=lfs merge=lfs -text
*.tiff filter=lfs diff=lfs merge=lfs -text
*.psd filter=lfs diff=lfs merge=lfs -text

# Video
*.mp4 filter=lfs diff=lfs merge=lfs -text
*.mov filter=lfs diff=lfs merge=lfs -text

# Unity specific bulky files
*.a filter=lfs diff=lfs merge=lfs -text
*.exr filter=lfs diff=lfs merge=lfs -text
*.pdf filter=lfs diff=lfs merge=lfs -text
*.cubemap filter=lfs diff=lfs merge=lfs -text
*.unity filter=lfs diff=lfs merge=lfs -text

```

---

## Step 3: Initialize the Local Repository

Now that our safety nets are in place, we can actually initialize Git.

1. Open your computer's **Terminal** or **Command Prompt**.
2. Navigate to your project folder. (Tip: You can usually type `cd ` and then drag and drop your project folder directly into the terminal window, then press Enter).
3. Run these commands one by one, pressing Enter after each:

```bash
# 1. Initialize the empty Git repository
git init

# 2. Activate Git LFS for this repository
git lfs install

# 3. Stage all your files (The .gitignore will automatically filter out the bad ones!)
git add .

# 4. Save this initial state
git commit -m "Initial commit with Unity setup and LFS"

```

---

## Step 4: Link to GitHub and Push

Your project is now safely version-controlled on your local computer. The final step is backing it up to GitHub.

1. Go to **GitHub.com** and log in.
2. Click the **`+`** icon in the top right and select **New repository**.
3. Give it a name (like `MyRetroHorrorGame`).
4. **CRITICAL:** Leave "Add a README", "Add .gitignore", and "Choose a license" **UNCHECKED**. You already made your own! We need a completely blank repository.
5. Click **Create repository**.
6. GitHub will show you a page with some terminal commands. Look for the section titled *"...or push an existing repository from the command line"*.
7. Copy those three lines of code. They will look exactly like this (but with your unique URL):

```bash
git remote add origin https://github.com/YourUsername/MyRetroHorrorGame.git
git branch -M main
git push -u origin main

```

8. Paste them into your Terminal and press Enter.

Git will now compress your code and upload your large assets through LFS directly to your GitHub repository. Depending on your internet speed and the size of your assets, this first push might take a minute or two.
